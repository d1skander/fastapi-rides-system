Работа с терминалом:

ls - показать список файлов в текущей папке.
cd имя_папки - зайти в папку.
cd .. - выйти на уровень выше.
mkdir имя_папки - создать новую папку.
ni .gitignore - быстро создать пустой файл.


Работа с git:
git status - проверить, какие файлы изменились.
git add . - подготовить все изменения к сохранению.
git commit -m "сообщение" - сохранить изменения локально.
git push origin main - отправить сохранения на GitHub.
git pull origin main - скачать обновления из GitHub в проект.
Remove-Item -Recurse -Force .git - полностью удалить историю Git.
git remote add origin <ссылка> - привязать папку к репозиторию.
git push -u origin main --force - принудительно перезаписать всё на GitHub.


Работа с uv:
uv python install - Установите версии Python.
uv python list - Просмотреть доступные версии Python.
uv python find - Найти установленную версию Python.
uv python pin - Закрепить текущий проект для использования определенной версии Python.
uv python uninstall - Удалите версию Python.
uv run - Запустите скрипт.
uv add --script - Добавить зависимость к скрипту.
uv remove --script - Удаление зависимости из скрипта.
uv init - Создайте новый проект на Python.
uv sync - Синхронизируйте зависимости проекта с окружением.
uv lock - Создайте файл блокировки для зависимостей проекта.
uv tree - Просмотрите дерево зависимостей проекта.
uv build - Встроить проект в архивы дистрибутива.
uv publish - Опубликовать проект в индексе пакетов.


Работа с alembic:
alembic init migrations — инициализация Alembic в проекте.
alembic revision --autogenerate -m "название" - автоматически создает файл миграции с описанием изменений.
alembic upgrade head - применить все последние изменения к базе данных
alembic upgrade +1 - применить только следующую по счету миграцию.
alembic downgrade -1 - откатить базу на одну миграцию назад.
alembic downgrade base - откатить базу в самое начало (удалить все таблицы, созданные через миграции).
alembic current - узнать, какая миграция сейчас применена к твоей базе данных.
